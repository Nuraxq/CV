In den folgenden Notebooks erklären wir, wie Sie die Computer-Vision-Umgebung einrichten, die für die Übungen verwenden werden. Die Umgebung besteht aus einem python-Interpreter, zusätzlichen Paketen für Bildverarbeitung, maschinelles Lernen und Visualisierung, einem jupyter-Notebook-Server sowie der Entwicklungsumgebung Visual Studio Code mit dem Versionsverwaltungssystem git ein (s. e00_vscode_git.ipynb)

Einrichtung von python and jupyter

Für die praktischen Übungen benötigen Sie eine funktionierende python-3.11.-Umgebung auf einem Betriebssystem Ihrer Wahl. Wir empfehlen die Installation der neuesten Version von Anaconda (Windows, macOS, Linux), einer Open-Source-Python-Distribution, die bereits für Aufgaben in der Computer Vision und Bildverarbeitung vorkonfiguriert ist. Folgen Sie der Installationsanweisung für Ihr Betriebssytem. Wir empfehlen zudem, mit virtuellen Umgebungen auf Basis von conda zu arbeiten.

Wir werden intensiv sogenannte Jupyter Notebooks verwenden, also interaktive, ausführbare Dokumente, die sich auch zur Erstellung von Berichten eignen.

Einrichten von conda

Überprüfen Sie zunächst, ob conda auf ihrem Rechner bereits installiert ist. Führen Sie dazu den folgenden Befehl im Terminal aus:

conda --version

Falls conda nicht verfügbar ist, installieren Sie Anaconda oder miniconda. Die betriebssystemspezifische Installation finden Sie im Miniconda Installation Guide.

Anschließend können Sie nun eine eigenen virtuelle Umgebung via conda erstellen und die benötigten Pakete manuell installieren. Für die Veranstaltung stellen wir eine environment.yml-Datei bereit, die automatisch eine passende Umgebung erstellt und die erforderlichen Pakete installiert. Im Verlauf der Veranstaltung kann es vorkommen, dass zusätzliche Pakete nachinstalliert werden müssen oder aktualisierte Umgebungen bereitgestellt werden.

Um die bereitgestellte Umgebung zu installieren, öffnen Sie ein Terminal und führen Sie den folgenden Befehl aus:

conda env create -f environment.yml

Überprüfen Sie anschließend die verfügbaren Umgebungen:

conda env list

Neben base sollte nun auch eine Umgebung namens cv_intro aufgeführt sein.

Nachdem Sie sich mit der Verwenung von jupyter vertraut gemacht haben, wählen Sie im jupyter-Notebook im Menü unter Kernel die Umgebung cv_intro aus. Das Importieren der benötigten Pakete sollte nun ohne Fehlermeldungen funktionieren.

Informationen zur Einrichtung von python und jupyter

Alternativ zur Anacoda Umgebung können Sie entweder mit einer reinen python-3.x-Installation arbeiten oder diese durch virtuelle Umgebungen (z.B. mit venv) ergänzen. In diesem Fall müssen Sie die benötigten Pakete manuell über den Python-Paketmanager pip installieren. Um eine Liste der installierten Python-Pakete anzuzeigen, können Sie ein Terminal öffnen und folgenden Befehl ausführen:

> pip list

Die für die Bearbeitung benötigten Pakete stellen wir in einer conda-Umgebung zur Verfügung. Möchten Sie die virtuelle Umgebung selber pflegen wollen, beachten Sie die folgenden Hinweise.

Verwenden Sie nach der Installation von Anaconda unter Windows das von Anaconda bereitgestellte Anaconda Prompt-Terminal. Dadurch wird sichergestellt, dass alle Umgebungsvariablen korrekt gesetzt sind. Außerdem empfiehlt es sich, zur Installation von Paketen die Eingabeaufforderung mit Administratorrechten zu öffnen (bzw. unter Unix-basierten Systemen su zu verwenden).

Für die praktischen Übungsaufgaben benötigen wir die folgenden zusätzlichen Pakete:

Verwendung von jupyter notebooks

Nachdem alle Pakete installiert wurden, sollten Sie in der Lage sein, einen Jupyter-Server zu starten. Dabei handelt es sich um einen Prozess, der Python-Code in sogenannten Kernels ausführt und die Ergebnisse zurückliefert. Dieser Jupyter-Server kann lokale Python-Kernels betreiben, die im Hintergrund als eigenständige Prozesse laufen. Sie empfangen Anfragen mit Python-Code, führen diesen aus und geben die Ergebnisse zurück. Eine verbreitete – wenn auch nicht die einzige – Möglichkeit, mit solchen Kernels zu interagieren, sind die browserbasierten Oberflächen jupyter notebook und jupyter lab.

Beide Oberflächen sind sich insofern ähnlich, als sie es ermöglichen, jupyter-Notebooks direkt im Browser zu öffnen, zu bearbeiten und auszuführen. Ein Notebook besteht aus mehreren Eingabe- und Ausgabezellen. In den Eingabezellen wird der auszuführende Code definiert, während die zugehörigen Ausgabezellen die Ergebnisse der Berechnung anzeigen. Wichtig ist, dass diese Zellen Code in verschiedenen Programmiersprachen wie python, julia oder R enthalten können (daher der Name jupyter), aber auch Markdown-formatierten Text, HTML-Code oder sogar LaTeX-Formeln. Dadurch eignen sich jupyter-Notebooks hervorragend zur Erstellung interaktiver, ausführbarer Dokumente, die sich beispielsweise für Computer-Vision-Experimente, Auswertungen und Berichte nutzen und direkt in HTML oder LaTeX/PDF exportieren lassen.

Für diesen Kurs können Sie entweder eine lokale Umgebung in einer IDE wie VS Code (s. e00_vscode) verwenden oder den Server jupyter notebook als auch jupyter lab, wobei letzterer die modernere Variante darstellt. Um einen jupyter lab-Server zu starten, navigieren Sie zunächst im Terminal in das Verzeichnis Ihres Dateisystems, in dem Sie ein jupyter-Notebook erstellen oder öffnen möchten, und führen Sie anschließend den folgenden Befehl aus:

> jupyter lab

Als ersten Schritt können Sie dies im Verzeichnis ausprobieren, das dieses Notebook enthält (d. h. die entsprechende .ipynb-Datei), s. auch e00_jupyter.html. Es öffnet sich ein Browserfenster, in dem Sie im linken Bereich den Dateibrowser sehen sollten. Dort wird das Notebook angezeigt – klicken Sie darauf, um es als interaktives Dokument zu öffnen. Auf ähnliche Weise können Sie auch ein neues Notebook erstellen. Durch Doppelklicken auf eine Eingabezelle können Sie den enthaltenen Text oder Code bearbeiten. Mit Shift+Enter führen Sie die Zelle aus; dabei wird entweder formatierter Text oder die Ausgabe des zugrunde liegenden Codes angezeigt.

Probieren wir dies in der folgenden Eingabezelle aus, die python-Code enthält und eine einfache textuelle Ausgabe erzeugt:

In [1]:
x = 2 * 22
print(x)
44

Sie können unterhalb der aktuellen Cursorposition eine neue Eingabezelle erstellen, indem Sie entweder auf die Schaltfläche + im Menü oben links klicken oder die Taste b auf der Tastatur drücken. Wenn Sie eine Zelle bearbeiten, können Sie mit Esc in den Kommando-Modus wechseln. In diesem Modus lassen sich Zellen hinzufügen, verschieben oder löschen. Zum Löschen einer Zelle drücken Sie im Kommando-Modus zweimal D. Um den Zelltyp von python auf markdown zu ändern, drücken Sie im Kommando-Modus m. Mit y wechseln Sie wieder zurück zu python-Code.

Probieren wir dies nun mit der folgenden Markdown-Zelle aus, die eine LaTeX-Formel enthält:

Hinweis: Wir können auch LaTeX-Formeln verwenden: ∫0πsin⁡(t) dt\int_0^\pi \sin(t) \, dt∫0π​sin(t)dt

Der gesamte Python-Code wird vom zugrunde liegenden Python-Kernel ausgeführt, dessen aktueller Status durch den Kreis oben rechts im Notebook-Fenster angezeigt wird. Ein nicht ausgefüllter Kreis (d. h. mit weißem Inneren) zeigt an, dass der Kernel momentan inaktiv ist. Ist der Kreis schwarz ausgefüllt, führt der Kernel gerade Berechnungen aus.

Dies können Sie beobachten, wenn Sie die folgende Zelle ausführen (Shift+Enter):

In [2]:
s = 0
for i in range(80000000):
    s += 1
print(s)
80000000
In [3]:
print(s)
80000000

Abschließend ist es wichtig zu beachten, dass der Python-Kernel lediglich ein einzelner Interpreter-Prozess ist, der den Code in den ausgeführten Zellen nacheinander abarbeitet. Das bedeutet, dass die Reihenfolge der Ausführung den aktuellen Zustand des Kernels bestimmt, also welche Variablen existieren und welche Werte sie besitzen.

Insbesondere bleibt der Zustand über mehrere Zellen hinweg erhalten, wie Sie im folgenden Beispiel sehen können:

In [4]:
x = 42
In [5]:
print(x)
42

Wenn Sie diese beiden Zellen in umgekehrter Reihenfolge ausführen, entsteht ein Fehler. Das wirkt zunächst trivial, kann jedoch bei komplexeren Notebooks, in denen Zellen mehrfach und in unterschiedlicher Reihenfolge ausgeführt werden, schnell unübersichtlich werden, da der aktuelle Zustand des Kernels nicht immer offensichtlich ist. Sie können den aktuellen Zustand jederzeit zurücksetzen, indem Sie den Kernel neu starten, wodurch ein neuer Interpreter-Prozess gestartet wird. Dies können Sie entweder über die Option „Restart Kernel“ im Menü Kernel tun oder indem Sie in den Kommando-Modus wechseln (ESC drücken) und anschließend zweimal die Taste 0 drücken. Probieren Sie dies aus und führen Sie danach die folgende Zelle aus – diese sollte einen Fehler erzeugen, da die Variable x im neuen Kernel noch nicht definiert wurde.

In [6]:
print(x)
42
